# Aura OS Architecture: The Business to Code Bridge

This document serves as the permanent memory blueprint for how the Business Dashboard interacts with the Sovereign Developer Studio (Aura Codes).

## 1. The Separation of Concerns
*   **Aura Codes (The Engine Room):** The Sovereign Developer Studio. This is where all source code is written, managed, built, and hosted. No coding happens in the Business OS.
*   **Master Dashboard / Port 5170 (The Driver's Seat):** The operational UI for the business owner. Bill uses this to manage staff, check GPS fleets, upload plans, and view analytics. It simply *consumes* the live URLs hosted by Aura Codes.

### Auto-Generated Base Projects
When a new business is created on the network, Aura Codes provisions exactly **two** base projects:
1. **The Internal POS Terminal:** The private cash-register application (`pos.aurame.ca`).
2. **The Public Website & Portals:** A single unified web application (`acmeplumbing.com`). The **Client Portal** and **Staff Portal** are NOT separate isolated websites; they are simply secure `/client-login` and `/staff-login` pages built directly into the main website via the Web Builder. 

*   These base projects are tied to the monthly Sovereign package. The business gets a simple, pre-made website template out-of-the-box.
*   If they want to upgrade to "God Mode", they can send the project over to their Pebble (Aura Codes) to unlock real live code injection, custom HTML, and Shopify-level architecture.
*   If the business owner wants to build a new custom tool (e.g., a "Disciplinary Form"), they build it in Aura Codes as an add-on project, which then gets hosted via Webhooks.

## 3. UI Routing & Navigation
To keep the Business Dashboard perfectly clean, navigation is strictly divided into two types:
*   **System Operations (Internal Inline Panels):** Settings like `Business Blueprint`, `POS Settings`, and `Pebble Settings` live natively inside Port 5170. Clicking them does not open a new tab; it simply triggers a smooth iPhone-style sliding animation to reveal the settings panel.
*   **Live Portals (External Links):** Web apps generated by Aura Codes like `Client Portal`, `True Oath / Staff`, and the `Public Website`. The Master Dashboard provides a simple button that links directly to the live hosted domain/port.

## 4. Webhooks & Billing
*   Because all domains and hosting are managed by Aura Codes, the system knows exactly what is hosted internally vs externally.
*   This allows the Master Dashboard to accurately reflect whether the business has paid their monthly fees for hosting and domain connections.
